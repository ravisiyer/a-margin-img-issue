Anchor margin and contained image max-width tests

  • CSS 'main-book img' selector has declaration: 'max-width: 90%;'
    Horizontal scrollbar appears only at very low window width (less than 200px) and so not an issue
  • CSS 'main-book img' selector has declaration: 'max-width: 100%;'
    Horizontal scrollbar appears at medium window width (less than 700px) and so an issue
  • CSS 'main-book img' selector has declaration: 'max-width: 100%;' and 'main-book a' selector sets margin to 0 and uses important
    Horizontal scrollbar appears only at very low window width (less than 200px) and so not an issue

I also tried out 100vw for img max-width. As expected, it did not solve the issue of horizontal scrollbar appearing at less than around 700px.

Removing the style statement from the anchor element in which the image is contained, also fixes the problem even if img max-width is 100%. But that involves editing the HTML file and so is not feasible as a solution for BFTB app. However, just for capturing that test, I have included the page.
